# 게임 재설계 문서

erAV_Ho v2.0 - 창관 경영 시뮬레이션

---

## 개요

**장르**: 창관 경영 + 육성 시뮬레이션
**목표**: 빚 청산 + 리그 승격
**수익**: 창관 운영(안정) / 조교 후 판매(한방)

**핵심 시스템**:
- D&D 주사위 판정
- 림월드식 랜덤 이벤트
- FM 리그 시즌제

---

## 시간 구조

### 하루 = 2행동 + 3체크

```
기상 체크 → 오전 행동 → 중간 체크 → 오후 행동 → 취침 체크
```

**체크 포인트**: 컨디션 변화, 이벤트 발생, 정산
**행동**: 조교 / 영업(오후만) / 시장 / 상점 / 휴식

**주기**: 1주=7일, 1시즌=12주, 주말 리그전, 월말 컵대회

## 주사위 시스템

### 주사위 종류
| 주사위 | 용도 |
|--------|------|
| d4~d12 | 효과량 (스탯 증가, 데미지 등) |
| d20 | **판정 (메인)** |
| d100 | 확률, 테이블 선택 |

### d20 판정
```
1d20 + 보정치 vs DC

보정치 = 캐릭터 스탯 + 조교사 스킬 + 도구 + 컨디션 + 특성

Natural 20: 자동 성공 + 보너스
Natural 1:  자동 실패 + 페널티

어드밴티지: 2d20 중 높은 값 (유리)
디스어드밴티지: 2d20 중 낮은 값 (불리)
```

### DC 기준
```
DC 5~10:  쉬움
DC 15:    보통
DC 20~25: 어려움
DC 30:    거의 불가능
```

### 수식 표기
```
1d20+5      d20 굴림 + 5
2d6+3       d6 2번 합계 + 3
4d6 drop 1  d6 4번, 최저 버림
```

## 캐릭터 시스템

### 기존 데이터 유지
```
abl (34), talent (261), exp (92), palam (17), mark
→ 복잡한 육성 깊이 유지
```

### 새 필드 추가
```typescript
tier: 'N' | 'R' | 'SR' | 'SSR'
stars: number                  // 각성 0~5

condition: {
  hp: number    // 체력 0~100
  mp: number    // 멘탈 0~100
  mood: number  // 기분 0~100
}

traits: string[]               // 림월드식 특성
assignment: string             // 배치 상태
baseValue: number              // 기본가
marketValue: number            // 시장가
```

### 티어
```
SSR: 500만, 70~100스탯, 3~4특성, 3%
SR:  200만, 50~80스탯,  2~3특성, 12%
R:   50만,  30~60스탯,  1~2특성, 35%
N:   10만,  10~40스탯,  0~1특성, 50%
```

### 컨디션
```
HP:   영업/조교 소모, 휴식 회복, 0이면 행동불가
MP:   가혹조교 감소, 칭찬 회복, 낮으면 반항/도주
MOOD: 80+ 어드밴티지, 30- 디스어드밴티지
```

### 특성 예시
```
긍정: 순종적(DC-3), 민감체질(경험치+50%), 인기인(지명+20%)
부정: 반항적(DC+5), 허약체질(HP-20%), 트라우마(특정거부)
특수: 마조(고통→쾌락), 새디(특수손님), 수집가인기(가치2배)
```

### 스탯 매핑
```
기존 abl/palam → 주사위 보정치 변환
예: abl[12] 기교 → 기교 보정 +3
```

## 조교 시스템

### 커맨드 & DC
```
기본 (DC 10~15): 복종, 체력, 매너, 외모
고급 (DC 15~20): 기교, 멘탈, 특수스킬
하드 (DC 20~25): 복종강화, 한계돌파, 각성
```

### 판정 흐름
```
커맨드 선택 → 보정치 계산 → d20 판정 → 결과 처리

성공:   스탯 + (1d4 + 보정)
실패:   효과없음, MP -5
Nat 20: 스탯↑↑, 스킬 습득
Nat 1:  역효과, 트라우마
```

### 도구
```
맨손:     +0
기본:     +1 (10만)
고급:     +2 (50만)
전문:     +3 (200만)
명품:     +5 (1000만)
```

## 창관 영업

### 손님 생성
```
손님 수 = 3 + 평판 + 1d4
등급: 일반/고급/VIP (리그별 비율)
선호: 매력/봉사/기교/순종
지불력: 1만~100만
DC: 10~25
```

### 서비스 판정
```
1d20 + 캐릭터(선호 스탯) vs 손님 DC

Nat 20: 수입×3, 팁, 단골, 평판↑
성공:   수입×1
실패:   수입×0.3, 평판↓
Nat 1:  트러블 (손해배상/부상/단속)
```

### 수입
```
기본 × 인기(0.8~1.5) × 리그(1.0~2.0) × 평판(0.9~1.3) + 팁
```

## 판매 시스템

### 가치 계산
```
기본가 × 레벨 × 스탯 × 스킬 × 특성 × 시장변동
```

### 구매자
```
일반:     시세, 항상
수집가:   ×1.5, 특정조건, 50%
귀족:     ×2~3, SR+, DC20협상, 30%
경쟁업소: ×0.7, 즉시, 20%
의뢰:     ×2+보너스, 스펙맞춤, 기한, 주1회
```

## 리그 시스템

### 4부 리그
```
1부 챔피언스: VIP50%, SSR10%, 유지50만, 수입×2.0
2부 프리미어: VIP20%, SSR5%,  유지30만, 수입×1.5
3부 챌린저:   VIP5%,  SSR1%,  유지15만, 수입×1.2
4부 루키:     VIP1%,  SSR0%,  유지5만,  수입×1.0 (시작)
```

### 시즌
```
12주 = 1시즌
주말: 리그전 정산 (수익40% + 평판30% + 품질20% + 대회10%)
월말: 컵 대회
시즌말: 상위2팀 승격, 하위2팀 강등
```

### 승점
```
1위: +3, 2~3위: +2, 4~6위: +1
컵우승: +2, 특별이벤트: +1
```

### 승강격
```
승격: 보너스자금, 슬롯+1, 콘텐츠해금
강등: 평판↓, VIP이탈, 콘텐츠잠김
```

## 이벤트 시스템

### 발생 구조
```
3체크마다 30% 확률 → 풀에서 랜덤 선택 → 조건 체크 → 실행
```

### 이벤트 풀
```
아침형 (70% 일반 / 20% 긍정 / 10% 부정)
  일반: 컨디션변화, 상인, 날씨
  긍정: 컨디션최고, 스카우트, 후원
  부정: 아픔, 도주시도, 악몽

낮형
  일반: 대화, 시장변동, 소문
  긍정: VIP예약, 후원, 평판↑
  부정: 충돌, 경쟁업소, 단속소문

밤형
  일반: 피드백, 소문, 피로
  긍정: 대만족, 입소문, 휴식효과
  부정: 트러블, 단속, 도둑
```

### 특수 이벤트
```
스토리: 조건달성, 캐릭터고유, 엔딩분기
대회:   그랑프리(월1회), 콘테스트
시즌:   크리스마스, 특별손님, 한정아이템
```

## 경제 시스템

### 난이도
```
EASY:   자금200만, 빚500만,  이자3%, 목표8시즌
NORMAL: 자금100만, 빚1000만, 이자5%, 목표12시즌
HARD:   자금50만,  빚2000만, 이자7%, 목표16시즌
```

### 수입
```
영업: 1만~100만 (등급별, 리그배율)
판매: 10만~3000만 (티어별)
대회: 100만~5000만
의뢰: 20만~500만
```

### 지출
```
고정: 유지비(5만~50만), 급여(5천~5만), 이자(주3~7%)
변동: 아이템, 도구, 치료비, 벌금, 손해배상
선택: 매입, 업그레이드, 광고
```

### 승패
```
패배: 파산(자금0 7일) / 몰락(캐릭터전멸) / 추방(최하위2시즌)
승리: 빚청산 + α (리그우승/자금1억)
```

### 시스템 간 연결 원칙

```
- 시스템끼리 직접 import 안 함 (스파게티 방지)
- 모두 GameState(Store)를 읽고/쓸 뿐 (유기적 연결)
- EventBus는 알림용 ("이런 일 발생" → 반응은 각자)

예: 조교로 스탯↑ → Store에 저장
    → 영업 시 Store에서 읽어서 판정
    → 자연스럽게 연결, 서로 모름
```
