# Phase 2. 핵심 골격 확장

범위: M7~M16

목표: 주요 기능을 한 단위씩 얇게 붙이고 저장/로드, 턴 진행, 검증 체계를 만든다.

완료 기준: 영입, 턴 종료, 저장/로드, 방문, 미션, 업무, 촬영, 훈련이 각각 최소 성공/실패/취소 또는 예외 경로를 갖고 build가 통과한다.

| M | 역할 | 책임 |
| --- | --- | --- |
| M7 | 고용/영입 1차 | 영입 listing, 돈 부족, 생성 결과, 중복 실패, 취소를 구현한다. |
| M8 | 턴 종료/시간 진행 | 턴 종료, 주차/월 롤오버, phase 복귀, session 폐기를 구현한다. |
| M9 | 저장/로드 | save payload schema, roundtrip, 손상/future/runtime 오염 실패를 구현한다. |
| M10 | 방문/시설 1차 | 방문 장소/행동 view와 기본 시설 사용 루프를 구현한다. |
| M11 | 미션 1차 | 미션 수락/보고/조건/보상 최소 루프를 구현한다. |
| M12 | 업무 1차 | 업무 선택, 인물 선택, 결과 계산, 턴 종료를 구현한다. |
| M13 | 촬영 1차 | 촬영 대상/장면 선택, 결과 계산, 수익/팬/경험 반영을 구현한다. |
| M14 | 훈련 1차 | 훈련 command 1개와 session 계산 buffer, 결과 반영을 구현한다. |
| M15 | 화면 정리/진단 | route별 화면 렌더링과 읽기 전용 진단 패널을 정리한다. |
| M16 | 테스트/검증 체계 | roundtrip, boundary, raw-name, stub gate를 만든다. |

주의: Phase 2는 기능 골격이다. 전수 구현 완료가 아니다.
