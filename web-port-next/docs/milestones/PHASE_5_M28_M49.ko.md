# Phase 5. 기능군별 전수 구현

범위: M28~M49

목표: 전수표를 기준으로 기능군별 원본 컨텐츠를 구현, 검증, 사용자 승인 제외로 닫는다. blocker는 완료 상태가 아니라 완료 차단 상태로만 둔다.

완료 기준: 각 기능군 종료 시 미구현 feature, 미소비 definition, 미정 save/session row, 소유 blocker가 해당 기능 범위에 남지 않는다.

| M | 역할 | 책임 |
| --- | --- | --- |
| M28 | 메인 route 연결자 | 원본 메인 화면 직속 route/action/menu/view row를 모두 연결한다. |
| M29 | 구매 상점 구현자 | 구매형 listing, 가격, 노출/해금, 구매 성공/실패/취소를 전수 구현한다. |
| M30 | 아이템 사용 구현자 | 사용형/특수 아이템, 시설 해금형, 장비/의복형 효과를 구현한다. |
| M31 | 영입 구현자 | 영입 listing 전체를 캐릭터 원형, 비용, 생성 결과와 연결한다. |
| M32 | 인물 identity 관리자 | Chara template과 이름/호칭/별명/표시 identity 경계를 닫는다. |
| M33 | 신체/능력/소질 관리자 | BASE/ABL/TALENT/EXP/MARK/PALAM 계열을 body/people owner로 분해한다. |
| M34 | 관계/CFLAG/장비 관리자 | CFLAG, 관계, 장비, 의복 상태를 의미별 owner로 분해한다. |
| M34.5 | gate hardening 관리자 | M35 이후 누락 방지 gate를 실제 script와 실패 조건으로 고정한다. |
| M35 | 턴/시간 관리자 | day/week/month/year, phase, hook 순서, 자동 처리, session cleanup을 구현한다. |
| M36 | 방문/시설 구현자 | 방문 장소, 장소별 행동, 시설 해금과 결과 반영을 전수 구현한다. |
| M37 | 업무/창관 구현자 | 업무/창관/아르바이트/특수 업무의 조건과 결과를 전수 구현한다. |
| M38 | 촬영 정의 관리자 | 촬영 장면 정의, 대상/장면 조건, 예상 결과를 구현한다. |
| M39 | 촬영 실행 관리자 | 촬영량, 수익, 팬, 점수, 이력, 출시/판매 상태를 구현한다. |
| M40 | 훈련 세션 관리자 | 훈련 대상/실행자/조수/command 선택과 session lifecycle을 완성한다. |
| M41 | 훈련 가능 조건 구현자 | 105개 command availability와 불가 사유를 전수 구현한다. |
| M42 | 훈련 0~34 효과 구현자 | command 0~34 원본 효과 계산과 결과 반영을 구현한다. 현재 blocked다. |
| M43 | 훈련 35~69 효과 구현자 | command 35~69 원본 효과 계산과 결과 반영을 구현한다. |
| M44 | 훈련 70~104/후처리 구현자 | command 70~104와 훈련 후처리, 이벤트/장비/자원 변화를 구현한다. |
| M45 | 공통 유지보수 구현자 | 능력 상승, 휴식, 회복, 자동 아이템 사용, 공통 시스템 흐름을 구현한다. |
| M46 | 미션 구현자 | 미션 정의/수락/진행/보고/완료/실패/만료/보상/패널티를 구현한다. |
| M47 | 세계/이벤트 구현자 | 이벤트 trigger/condition/text/effect, world/story flag, hook을 구현한다. |
| M48 | 엔딩/meta 구현자 | 엔딩 조건/결과, 계승, global/meta save, 전역 해금을 구현한다. |
| M49 | 정보/도움말/설정/view/text 정리자 | 정보/도움말/설정/디버그, text, 남은 기능 감사를 닫는다. |

주의: 샘플 구현, no-op handler, 표시만 되는 화면, static profile 생성은 전수 구현 완료가 아니다.
