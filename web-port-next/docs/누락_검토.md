# 누락 항목 검토 결과

작성일: 2026-05-25

범위: `web-port-next/docs/누락.md`의 모든 P0/P1/P2 항목을 원본 ERB와 현재 구현으로 재검토했다.

## 결론

대부분은 실제 이슈다. 다만 세 항목은 오탐 또는 플레이어 UX 이슈로 보기 어려워 `누락.md`에서 제거하거나 재작성했다.

- 삭제: `저장/로드 피드백 문자열 깨짐`
  - UTF-8 기준 `web-port-next/src/features/saveLoad.ts`의 저장/로드 성공 메시지는 정상 한국어다.
- 삭제: `영입 페이지 이동이 실제 후보 수와 분리됨`
  - 원본도 영입 페이지를 0~3 고정 범위로 운용한다.
- 삭제: `shop/selectItem action은 있으나 UI에서 호출되지 않음`
  - UI는 `shop/selectListing`으로 같은 선택 상태를 세팅한다. 내부 cleanup 후보일 수는 있지만 원본 UX 누락은 아니다.
- 재작성: `immediate-use catalog와 보유 아이템 사용 UX가 섞임`
  - 원본 immediate-use도 보유품 소비가 아니라 구매 직후 사용이다. 실제 문제는 현재 UI 라벨/배치가 보유품 사용처럼 보일 수 있다는 점이다.

## 섹션별 판정

| 섹션 | 판정 | 보정 사항 |
|---|---|---|
| 1. 새 게임 / 부트 | 전부 실제 이슈 | P0 유지. 새 게임은 원본의 경고, 모드 설명, NTR 설정, 클리어 보너스, 오프닝 선택을 건너뛴다. |
| 2. 메인 메뉴 / SHOP | 대부분 실제 이슈 | 저장 payload 복원은 “원본 route metadata 누락”이 아니라 “현재 load가 항상 mainMenu session을 새로 만든다”로 이해해야 한다. 저장 피드백 깨짐은 삭제했다. |
| 3. 전역 결과 화면 | 전부 실제 이슈 | `result` route 미렌더링과 effect log 축약은 유지한다. |
| 4. 아이템 | 부분 구현 + 원본 UX 누락 | 구매/수량/사용은 기능 일부가 있으나 원본의 yes/no, 입력 루프, item별 성공 `WAIT`가 없다. immediate-use 항목은 P2 라벨 문제로 낮췄다. |
| 5. 인재 영입 | 계약 후 UX는 실제 이슈 | 페이지 고정 문제는 원본 parity라 삭제했다. |
| 6. 로스터 | 전부 실제 이슈 | `SHOW_CHALADATA`, `ACTRESS_LIST`, `ACTRESS_PROF`가 하나의 roster로 합쳐진 점은 유지한다. |
| 7. 정렬/부탁/기숙사 | 전부 실제 이슈 | P0/P1 유지. 독립 route/action/session이 없다. |
| 8. 의상 | 전부 실제 이슈 | 현재 wardrobe가 얕은 대체 구현이다. P1 유지. |
| 9. 방문 | 전부 실제 이슈 | 일부는 flat action list로 대체됐으나 원본 하위 메뉴/대상 선택/확인 루프가 없다. |
| 10. 창관 업무 | 전부 실제 이슈 | `YUUKAKU_TOP` 자체는 없지만 generic work route는 있으므로 “기능 전체 누락”이 아니라 “상위 메뉴 UX 누락”으로 읽어야 한다. |
| 11. AV 촬영 | 전부 실제 이슈 | 준비 순서, 결과 화면, 후보 필터, 시간 이중 진행 위험 모두 유지한다. |
| 12. 조교 / 훈련 | 전부 실제 이슈 | 몇 항목은 부분 구현이 있으나 원본 session/result/aftertrain 구조가 없다. |
| 13. 미션 | 전부 실제 이슈 | 보고 버튼 조건 불일치는 실제 버그성 UX 이슈다. |
| 14. 턴 종료 / 이벤트 | 전부 실제 이슈 | 현재 일부 hook order는 문자열/효과로 존재하지만 화면 이벤트로 노출되지 않는다. |
| 15. 엔딩 / 강하게 뉴게임 | 전부 실제 이슈 | P0 유지. route/action/session이 없다. |
| 16. 설정 / 정보 / 도움말 / 실적 / 디버그 | 전부 실제 이슈 | 정보/TIPS/실적/cheat/debug는 실제 route가 없다. 여배우명부는 roster로 대체된 별도 문제다. |
| 17. 현재 UI 공통 품질 | 전부 실제 이슈 | 일부 목록에서 unavailable 항목이 클릭 가능하다. 생성 데이터 entry를 runtime catalog/view에 실을 곳이 없다. |

## Severity 보정 메모

- `창관 상위 메뉴 누락`은 P0로 유지할 수 있다. 다만 의미는 “업무 기능 전체가 없음”이 아니라 “원본 `YUUKAKU_TOP` 상위 화면이 없음”이다.
- `정보/TIPS/실적/cheat/debug 화면 누락`은 P0로 유지할 수 있다. 단 `700` 여배우명부는 완전 부재가 아니라 roster로 잘못 합쳐진 항목이다.
- `debug 편집기 누락`은 개발용 화면이므로 P1 유지가 맞고, P0까지 올릴 필요는 없다.
- `보고 버튼 상태와 report 처리 조건 불일치`는 현재 진행을 실제로 막을 수 있어 P1보다 높게 볼 여지가 있다. 다만 미션 전체 UX 누락과 함께 묶어 P1로 유지했다.

## 다음 구현 시 주의

원본 UX를 복구할 때는 단일 React 화면에 버튼만 추가하는 방식으로 닫으면 안 된다. 대부분의 누락은 다음 중 하나다.

- 원본의 `INPUT`/`INPUTS` 단계가 현재 session state로 보존되지 않는다.
- 원본의 `WAIT` 결과 화면이 effect log로 흡수된다.
- 원본의 상위 메뉴와 하위 메뉴가 현재 flat list로 합쳐졌다.
- 원본의 자동 이벤트가 state mutation으로만 처리되고 플레이어 화면으로 나오지 않는다.
- 원본의 page/list/previous/next/return 흐름이 React local state로만 유지된다.
