# 누락 화면 흐름 / UX 감사

작성일: 2026-05-25

범위: `original-game/ERB`의 원본 화면/입력 흐름과 `web-port-next`의 현재 React route/session 흐름을 대조했다. 세부 구현 로직의 수치 일치 여부가 아니라, 플레이어가 실제로 보게 되는 화면 순서, 선택지, 확인 루프, 결과 표시, 복귀 흐름이 원본 대비 빠졌거나 축약된 지점을 기록한다.

검토 결과: `web-port-next/docs/누락_검토.md`

## 판단 기준

- **P0 - 화면 자체가 없음**: 원본에는 독립 메뉴, 하위 화면, 입력 루프가 있으나 현재 route/action/view가 없거나 disabled 계약만 있다.
- **P1 - 화면은 있으나 원본 UX가 축약됨**: 현재 화면에서 기능은 일부 실행되지만 원본의 확인, 대상 선택, 실패 메시지, 결과 `WAIT`, 반복 입력, 하위 메뉴가 빠졌다.
- **P2 - 표시/상태 UX 품질 누락**: 핵심 진행은 가능하더라도 원본이 보여주던 상태, 설명, 도움말, 경고, 세부 프로필, 저장/복귀 맥락이 빠졌다.

근거는 원본 ERB, 현재 코드, 생성 데이터에 남은 line anchor를 함께 적었다.

## 전역 결론

현재 `web-port-next`의 UI route는 `boot`, `newGame`, `mainMenu`, `itemShop`, `mission`, `recruit`, `visit`, `work`, `shooting`, `roster`, `wardrobe`, `training`, `result`, `saveLoad`로 제한된다. 이 중 `result`는 타입에는 있지만 실제 렌더 분기가 없어 unknown route로 떨어진다. 근거: `web-port-next/src/game/routes.ts:14`, `web-port-next/src/ui/RouteScreens.tsx:1286`.

원본은 거의 모든 주요 행동 뒤에 전용 출력, 확인, `WAIT`, 재입력 루프, 하위 메뉴 복귀가 있다. 현재는 많은 결과가 `effects` 로그 또는 같은 화면의 간단한 상태 갱신으로 흡수된다. 그런데 effect 로그는 진단 패널을 켜야 보이는 구조라 일반 플레이 UX에서는 성공/실패/사후 메시지가 사라진 것처럼 보일 수 있다. 근거: `web-port-next/src/ui/AppLayout.tsx:30`, `web-port-next/src/ui/DiagnosticsPanel.tsx:188`.

메인 메뉴 기준으로 원본 24개 선택지 중 실제 화면으로 이어지지 않는 disabled route가 남아 있다. `data/coverage/main-route-coverage.json`은 disabled menu row 11개를 기록한다. 따라서 현재 메인 route 연결만으로 원본 화면 흐름 완료로 볼 수 없다.

- 현재: `web-port-next/src/catalog/types.ts:128`, `web-port-next/src/catalog/legacyCatalog.ts:511`, `web-port-next/src/game/routes.ts:1`

## 우선 구현 순서 제안

1. **공통 결과/확인 route부터 복구**: `result` route를 실제 렌더링하고, 구매/사용/방문/업무/촬영/훈련/미션/은퇴 결과를 일반 플레이어가 볼 수 있는 화면으로 올린다.
2. **새 게임 원본 순서 복구**: 주의문구, 모드 설명, NTR 설정, 클리어 보너스, 오프닝 선택을 `boot/newGame` flow로 분리한다.
3. **메인 disabled route 처리**: 설정, 정보, TIPS, 엔딩, 능력치업처럼 원본 독립 화면이 있는 항목은 disabled 버튼이 아니라 최소 원본형 화면 shell을 만든다.
4. **평평해진 화면을 원본 하위 메뉴로 재분해**: 방문, 업무, 촬영, 훈련은 현재 단일 카드/리스트 화면에서 원본의 대상 선택, 하위 메뉴, 확인, 결과, 복귀 루프를 되살려야 한다.
5. **로스터 계열 분리**: 능력 표시, 여배우 명부, 보유 캐릭터 관리, 매각/은퇴를 각각 다른 원본 UX로 분리한다.
6. **원본 입력/출력 단위로 완료 판정**: UX 화면 완료 판정은 “route 있음”이나 데이터 row 소비가 아니라 원본의 입력, 출력, 확인, 복귀, 효과, 저장/로드 복원까지 같은 단위로 연결됐을 때만 닫는다.
